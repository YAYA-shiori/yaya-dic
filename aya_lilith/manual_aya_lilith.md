# あやりりす 利用マニュアル

あやりりす（aya_lilith.dic）は、SSPのSHIORIイベントを日本語の関数名で書けるようにするライブラリです。

ゴースト作者は「起動」「終了」「なでなで0頭」のような日本語の関数名でトークを書くだけで、SSPのイベント処理が自動的に行われます。

## ランダムトーク

一定間隔で自動的に喋るトークです。

### 基本の書き方

```yaya
ランダムトーク通常 : array
{
	"\0\s[5]今日もいい天気だね。\w9\w9\1\s[10]そうだね。\e"

	"\0\s[6]お腹すいたなあ…\w5\w9\w9\1\s[10]ごはん食べよう。\e"
}
```

**重要：**
- 関数名の末尾に必ず `: array` を付ける
- 各トークは文字列1つにつき1つ

### 条件別ランダムトーク

時間帯・季節・曜日などに応じて、条件に合うトークだけが候補に加わります。

```yaya
// 季節別
ランダムトーク春 : array { ... }
ランダムトーク夏 : array { ... }
ランダムトーク秋 : array { ... }
ランダムトーク冬 : array { ... }

// 時間帯別（朝/昼/夕方/夜/深夜）
ランダムトーク朝 : array { ... }
ランダムトーク深夜 : array { ... }

// 曜日別
ランダムトーク月曜 : array { ... }
ランダムトーク土曜 : array { ... }

// 平日・休日
ランダムトーク平日 : array { ... }
ランダムトーク休日 : array { ... }

// 組み合わせ
ランダムトーク平日の朝 : array { ... }
ランダムトーク春の朝 : array { ... }
ランダムトーク休日の夜 : array { ... }
ランダムトーク冬の休日 : array { ... }

// 月別
ランダムトーク3月 : array { ... }
ランダムトーク12月上旬 : array { ... }

// 特定の曜日
ランダムトーク第2週土曜 : array { ... }

// 食事時（朝食時=7-8時、昼食時=12時、夕食時=18-19時）
ランダムトーク朝食時 : array { ... }

// 時間指定
ランダムトーク15時 : array { ... }
```

使えるスロット名の全リスト：

| カテゴリ | 例 |
|:--|:--|
| 時間帯 | `朝`, `昼`, `夕方`, `夜`, `深夜` |
| 時刻 | `0時` ～ `23時` |
| 季節 | `春`(3-5月), `夏`(6-8月), `秋`(9-11月), `冬`(12-2月) |
| 季節の時間帯 | `春の朝`, `冬の深夜` など |
| 食事時 | `朝食時`, `昼食時`, `夕食時` |
| 月 | `1月` ～ `12月` |
| 月旬 | `3月上旬`, `7月中旬`, `12月下旬` |
| 曜日 | `日曜` ～ `土曜` |
| 第N週曜日 | `第1週月曜`, `第3週金曜` |
| 行事 | `正月`(1/1-3), `クリスマス`(12/24-25), `年末`(12/26-31), `仕事`(それ以外) |
| 行事+曜日 | `仕事月曜`, `正月日曜` |
| 平日/休日 | `平日`(月-金), `休日`(土日) |
| 行事+平休 | `仕事平日`, `仕事休日` |
| 季節+平休 | `春の平日`, `冬の休日` |
| 平休+時間帯 | `平日の朝`, `休日の夜` |
| シェル | `シェル(シェル名)` |
| バルーン | `バルーン(バルーン名)` |
| 着せ替え | `着せ替えパーツ(名前)`, `着せ替えカテゴリ(名前)` |
| ユーザー定義 | `aya_lilith_config.dic` で追加可能 |

**条件の仕組み：** 通常トーク＋条件に合う全スロットのトークがまとめてシャッフルされ、その中からランダムに1つ選ばれます。条件別トークが多いほど、その条件の時に出やすくなります。

### カスタムトーク

通常の条件スロットとは別枠で、常に候補に含まれるトーク：

```yaya
ランダムトークカスタム : array { ... }
```

## 起動・終了トーク

### 基本

```yaya
// 通常の起動トーク（条件別がない場合のフォールバック）
起動
{
	"\0\s[5]こんにちは、\w5%(username)。\w9\w9\1\s[10]こんにちはー。\e"
}

// 通常の終了トーク
終了
{
	"\0\s[5]またね、\w5%(username)。\w9\w9\1\s[10]ばいばい。\e"
}
```

### 初回起動

```yaya
初回起動
{
	"\0\s[5]はじめまして。\w9\w9\1\s[10]よろしくー。\e"
}

// 消滅（Vanish）後の再インストール起動
// reference[0] に消滅回数が入る
Vanish後初回起動
{
	"\0\s[5]また会えたね。\e"
}
```

### 条件別起動・終了

ランダムトークと同じスロット名が使えます。

```yaya
// 時間帯別
起動朝 { ... }
起動深夜 { ... }
終了夜 { ... }

// 季節別
起動春 { ... }

// 組み合わせ
起動平日の朝 { ... }
春の朝に起動 { ... }  // 「～に起動」の形も可

// 特定日
起動3月14日 { ... }	   // 数字始まりは「起動」を先頭に
起動10月第2週土曜 { ... }

// カスタム（条件別より優先）
起動カスタム { ... }

// ユーザー誕生日
起動ユーザー誕生日 { ... }
```

**特定日トークの仕様：** 同じ日に複数回起動/終了しても、毎回違うトークが出るよう配列から順に消費されます。全て出尽くすと通常の起動/終了トークに戻ります。

## 時報

毎時0分に呼ばれます。

```yaya
// 基本（時刻指定）
時報0時 { ... }
時報12時 { ... }
時報23時 { ... }

// 特定日の特定時刻
時報3月14日12時 { ... }

// 全時刻共通（個別の時報がない場合のフォールバック）
時報 { ... }
```

探索順序：`時報M月D日H時` → `時報H時` → `時報`

### 日付変更

毎日最初の0分に1回だけ呼ばれます。

```yaya
日付変更 { ... }
日付変更1月1日 { ... }	// 特定日
```

### 累積起動時間

累積起動時間が1時間単位で変化した時に呼ばれます。

```yaya
累積起動時間変化
{
	// _argv[0] に累積時間（時間単位）が入る
}
```

## マウス反応

### 関数名の規則

```
[種別] + [スコープ番号] + [当たり判定名]
```

### 種別一覧

| 種別 | 説明 |
|:--|:--|
| `なでなで` | マウスを当たり判定上で動かす |
| `クリック` | 左クリック |
| `ダブルクリック` | 左ダブルクリック |
| `多重クリック` | 3回以上の連続クリック |
| `右クリック` | 右クリック |
| `右ダブルクリック` | 右ダブルクリック |
| `中クリック` | 中クリック |
| `ころころ上` | ホイール上回転 |
| `ころころ下` | ホイール下回転 |
| `ころころ` | ホイール回転（上下区別なし） |
| `ホールド` | マウスボタン長押し（1秒以上） |
| `ホールド終了` | 長押しを離した |
| `ドラッグ開始` | ドラッグ開始 |
| `ドラッグ終了` | ドラッグ終了 |
| `マウスホバー` | マウスを当たり判定上で静止 |
| `トーク中～` | トーク表示中の反応（通常より優先） |

### 書き方の例

```yaya
// \0の頭をなでなで
なでなで0Head
{
	"\0\s[5]なでなでされちゃった。\w9\w9\1\s[10]いいなー。\e"

	"\0\s[3]えへへ…\w5\e"
}

// \0の顔をダブルクリック
ダブルクリック0Face { ... }

// \1をなでなで（当たり判定名なし＝どこでも反応）
なでなで1 { ... }

// トーク中に\0の頭をダブルクリック
トーク中ダブルクリック0Head { ... }

// \0の胸を長押し
ホールド0Bust { ... }

// \0でホイール上回転
ころころ上0Head { ... }

// \1でホイール（上下区別なし）
ころころ1 { ... }
```

### 便利な変数

| 変数 | 説明 |
|:--|:--|
| `マウスイベント連続回数` | 3秒以内に同じ操作が繰り返された回数 |
| `クリック連続回数` | クリックの連続回数（ダブルクリック=2、多重=N） |
| `マウスイベントキャラID` | イベントが起きたキャラのスコープ番号 |

### マウスジェスチャー

マウスボタンを押しながら方向入力した時の反応です。

```yaya
// 方向の組み合わせ（「・」区切り）
マウスジェスチャー0Head上・下・左 { ... }

// 単発方向（移動のたびに呼ばれる）
マウスジェスチャー0Head単発上 { ... }

// 回転ジェスチャー
マウスジェスチャー回転0Head時計回り・時計回り { ... }

// 開始・終了
マウスジェスチャー0Head開始 { ... }
マウスジェスチャー0終了 { ... }
```

## ゴースト切り替え

### 関数名の規則

```
[ゴースト名] + [イベント名]
```

| イベント名 | 意味 |
|:--|:--|
| `へ変更` | そのゴーストに切り替える時（去る側の反応） |
| `から変更` | そのゴーストから切り替わってきた時（来る側の反応） |
| `を呼び出し` | そのゴーストを呼び出す時 |
| `から呼び出される` | そのゴーストに呼び出された時 |
| `を呼び出し完了` | 呼び出しが完了した時 |
| `が終了した` | そのゴーストが終了した時 |

### 書き方の例

```yaya
// 特定のゴースト（Sakura名 or フルネーム）
＊Emilyへ変更
{
	"\0\s[5]Emilyさんと交代します。\w9\w9\1\s[10]ばいばい。\e"
}

＊Emilyから変更
{
	"\0\s[5]Emilyさんから交代しました。\w9\w9\1\s[10]こうたいー。\e"
}

// 自分自身への切り替え
自分へ変更 { ... }
自分から変更 { ... }

// 個別に関数がないゴースト用のフォールバック
その他へ変更 { ... }
その他から変更 { ... }
他のゴーストへ変更 { ... }	// 「その他」と同じ
```

**ゴースト名の注意：**
- 数字で始まるゴースト名は先頭に `＊` を付ける（例：`＊54を呼び出し`）
- `+`, `-`, `"`, `'`, スペースなどの記号は `_` に置換される（例：`the "MobileMaster"` → `the__MobileMaster_`）

## コミュニケート

他のゴーストやユーザーからの話しかけに応答します。

### ユーザーからの話しかけ

```yaya
// キーワードマッチ
ユーザーコミュこんにちは { ... }

// OR条件（「｜」区切り）
ユーザーコミュ好き｜愛し { ... }

// AND＋OR条件（「＆」区切りが上位）
ユーザーコミュりりす＆好物｜好きなもの { ... }
// → 「りりす」を含み、かつ「好物」か「好きなもの」を含む時

// 該当なし（どのキーワードにもマッチしない時）
ユーザーコミュ該当なし { ... }
```

### ゴーストからの話しかけ

```yaya
// 特定ゴーストからの話しかけ
＊Emilyコミュこんにちは { ... }
＊Emilyコミュ該当なし { ... }

// どのゴーストからでも（フォールバック）
＊その他コミュこんにちは｜こんばんは { ... }
＊その他コミュ該当なし { ... }
```

### こちらから話しかける

```yaya
// 特定ゴーストに話しかける
＊Emilyに話しかける { ... }

// 汎用
その他に話しかける { ... }

// 起動中のゴーストがいない時
話しかけられない { ... }
```

メニュー等から `他に話しかける` を呼ぶと、起動中のゴーストからランダムに1体を選んで話しかけます。

## メニュー

`メニュー()` や `半分メニュー()` のユーティリティ関数でメニュー項目を生成します。あやりりすEXを使う場合は `＠メニュー` 記法も利用できます。

```yaya
OpenMenu
{
	"\0\s[5]メニューです。\w9\n"
	--
	半分メニュー('なにか話して', 'Menu_AITALK')
	--
	半分メニュー('名前を覚えて', 'Menu_TELLNAME')
	--
	"\n\n[half]"
	--
	メニュー('なんでもない', 'Menu_CANCEL')
	--
	"\e"
}

// メニュー選択時に呼ばれる関数
Menu_AITALK
{
	"\![raise,OnAITalk]"
}

Menu_CANCEL
{
	"\0\s[6]キャンセルね。\e"
}
```

### 選択肢タイムアウト

```yaya
// 選択肢を放置した時
選択肢時間切れ { ... }

// 特定のイベントの選択肢が時間切れした時
選択肢時間切れ(元イベント名) { ... }
```

同様に `バルーン時間切れ`、`バルーン中断`、`バルーン閉` も使えます。

## 名前入力

ユーザーの名前を入力してもらう機能です。

### 起動方法

```yaya
// メニュー等から呼ぶ
Menu_TELLNAME
{
	名前入力起動('メニュー')
}
```

### 要実装関数

`名前入力起動` の引数が識別子になり、以下の関数名の末尾に付きます。

```yaya
// 入力ボックスを開く時の台詞
名前入力開くメニュー
{
	"\0\s[5]じゃあ、\w5名前を教えてちょうだい。\w9\w9\1\s[10]敬称はなしでねー。\e"
}

// 空欄で確定された時
名前入力空欄メニュー { ... }

// 敬称付きで入力された時（「さん」付き等）
名前入力敬称メニュー { ... }

// 正常に名前が決まった時
名前入力終了メニュー
{
	"\0\s[5]わかった。\w9%(username)ね。\w9\w9\1\s[10]よろしくー。\e"
}
```

### 設定

`aya_lilith_config.dic` で設定：

```yaya
名前入力敬称 { 'さん' }		// 自動付加される敬称
名前入力初期 { 'ユーザ' }	// 初期ユーザー名（ゴーストに合わせて変更）
```

`username` 変数にユーザー名が格納されます。トーク内では `%(username)` で参照できます。

```yaya
見切れ0 { ... }		   // \0が画面外に出た

見切れ1 { ... }		   // \1が画面外に出た

```

## キー入力

特定キーが押された時の反応です。

```yaya
キー入力T { ... }		   // Tキー
キー入力連続T { ... }	   // Tキーの連続押し
キー入力SPACE { ... }	   // スペースキー
```

キー名は大文字で指定します。

**組み込みデバッグ機能：** Shift+D でデバッグメニューが開きます。`OnAYLXX_Debug_` で始まる関数が自動的にメニューに列挙されます。

## リンクメニュー

右クリックメニューに表示されるリンクです。

```yaya
// ポータルサイト
ポータルサイト
{
	"SSP BUGTRAQ | http://ssp.shillest.net/ |\n"
	--"整備班 | http://ms.shillest.net/ |"
}

// \0側おすすめサイト
おすすめ0
{
	"サイト名 | URL |"
}

// \1側おすすめサイト
おすすめ1 { ... }

// リンク選択時のトーク（任意）
ポータルサイトSSP_BUGTRAQ { ... }
おすすめ0(サイト名) { ... }
```

サイト名の記号は `_` に置換されます（関数名の制約）。

### フック関数

特定の名前で始まる関数を定義すると、あやりりすが自動で呼び出します。

| 関数名パターン | タイミング |
|:--|:--|
| `ゴースト起動前の処理*` | DLL読み込み時 |
| `ゴースト終了前の処理*` | DLL解放時 |
| `起動共通スクリプト*` | 起動トークの前に毎回付加 |
| `終了共通スクリプト*` | 終了トークの前に毎回付加 |
| `起動初期化スクリプト` | 最初のスクリプト出力時に1回だけ |

`*` の部分は任意の文字列で、`GETFUNCLIST` で自動列挙されます。

## ユーティリティ関数

トーク内や他の関数から呼び出せる便利関数です。

| 関数 | 説明 |
|:--|:--|
| `username` | 現在のユーザー名（変数） |
| `ウエイト消去(text)` | テキストから `\w` タグを除去 |
| `さくらスクリプト除去(text)` | 全タグを除去してプレーンテキストに |
| `さくらスクリプト無効化(text)` | タグをエスケープして表示用に |
| `エスケープ(text)` | SakuraScript選択肢用エスケープ |
| `累積起動時間` | 累積起動時間（時間単位） |
| `累積起動回数` | 累積起動回数 |
| `ゴースト起動時間` | 今回の連続起動時間（分単位） |
| `プラグインの存在(名前orID)` | プラグイン存在チェック（0 or 1） |
| `他に話しかける` | ランダムな起動中ゴーストに話しかける |
| `メニュー(表示テキスト, イベント名, ...)` | メニュー項目を生成 |
