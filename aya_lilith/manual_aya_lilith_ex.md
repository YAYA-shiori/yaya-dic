# あやりりすEX 利用マニュアル

あやりりすEX（aya_lilith_ex.dic）は、トークを簡易記法で書けるようにする変換エンジンです。SakuraScriptの `\0\s[5]`, `\1\s[10]`, `\n`, `\w9` 等を手書きする必要がなくなります。

## 基本の書き方

```
ヘッダ + サーフィスID + 区切り文字 + テキスト
```

- **ヘッダ** — キャラクターを識別する1文字。設定ファイルで定義
- **サーフィスID** — 表情番号（全角・半角どちらでも可）
- **区切り文字** — `：`（全角コロン）または `「」`（カギカッコ）

### ヘッダの設定例

ヘッダは `aya_lilith_ex_config.dic` でゴーストごとに設定します。以下はサンプルとして `エ`/`テ` を使用しますが、ゴーストのキャラクター名に合わせて自由に変更してください。

| ヘッダ | スコープ | 説明 |
|:--|:--|:--|
| `エ` | \0 | サンプル用 |
| `テ` | \1 | サンプル用 |

### 例

```yaya
<<"
エ０５：今日もいい天気だね。
テ１０：そうだね。
">>
```

これが自動的に以下のSakuraScriptに変換されます：

```
\0\s[5]今日もいい天気だね。\w9\w9\1\s[10]そうだね。\e
```

### カギカッコ記法

`：` の代わりに `「」` で囲むこともできます。

```yaya
エ０５「今日もいい天気だね！」
```

## ヒアドキュメント

トークはYAYAのヒアドキュメント構文で囲みます。

### `<<" ">>` — 通常用

YAYA変数 `%(...)` が**展開されます**。`<<" ">>` 内ではYAYAの変数・式が使えます。

```yaya
起動
{
	<<"
	エ０５：%(username)、こんにちは。
	">>
}
```

### `<<' '>>` — ランダムトーク配列用

YAYA変数は**展開されません**（高速化のため）。ただし `（ユーザー）` 記法は使えます。

```yaya
ランダムトーク通常 : array
{
	<<'
	エ０５：（ユーザー）、元気？
	テ１０：元気そうだね。
	'>>
}
```

**使い分けの目安：**
- ランダムトーク → `<<' '>>` を使う（トークが多い場合の速度対策）
- それ以外（起動・終了・メニュー・マウス反応等） → `<<" ">>` を使う

## 複数行の会話

### 同じキャラの連続発言

サーフィスを変えない場合は、ヘッダとサーフィスIDを省略できます。

```yaya
エ０５：1行目のテキスト。
：2行目のテキスト。
：3行目のテキスト。
```

`：` だけの行は前の発言者の続きになり、改行 `\n` が自動挿入されます。

### サーフィスだけ変える

ヘッダだけ書いてサーフィスIDを変えると、同じキャラで表情が変わります。

```yaya
エ０５：楽しいね！
エ０４：…でもちょっと疲れたかも。
```

### サーフィス省略

ヘッダだけ書いてサーフィスIDを省略すると、前のサーフィスを引き継ぎます。

```yaya
エ０５：楽しいね！
エ：ほんとに楽しい！
```

### キャラ切り替え

別のヘッダを書くとキャラが切り替わります。自動的にスコープ変更 + 改行 + ウエイトが入ります。

```yaya
エ０５：こんにちは！
テ１０：こんにちはー。
エ０４：ちょっと緊張するね。
```

### カギカッコの複数行

```yaya
エ０５「1行目
2行目
3行目」
```

開きカッコ `「` から閉じカッコ `」` までが1つの発言になります。

## サーフィス指定

### 基本

全角・半角数字どちらでも使えます。

```yaya
エ０５：テキスト	 // surface5
エ05：テキスト		 // 半角でもOK
エ５：テキスト		 // 1桁でもOK
```

### 非表示

```yaya
エ消：テキスト		 // \s[-1]（非表示）
```

### サーフィス変換

設定ファイルの `あやりりすEXSurfaceTranslate` で、番号の自動修正ができます。

設定例（\0がsurface0-9、\1がsurface10-19を使うゴーストの場合）：
- \0側にsurface10-19を指定 → 自動で-10される（`エ１２` → surface2）
- \1側にsurface0-9を指定 → 自動で+10される（`テ０２` → surface12）

うっかりスコープを間違えても自動修正されます。設定方法は[設定ファイル](#設定ファイルaya_lilith_ex_configdic)の項を参照。

## \iタグ（アニメーション発動）

サーフィスIDの後に `.`（全角 `．` でも可）で区切って番号を書くと、\sタグの後に\iタグで指定IDのアニメーションを発動できます。

```yaya
エ０５．２０２２：テキスト	  // \s[5]\i[2022]
テ１０．２０５０：テキスト	  // \s[10]\i[2050]
```

吹き出しの後ろに `バ` (または `b`/`ｂ`) + 番号で `\b[N]` タグも付加できます。

```yaya
エ０５バ３：テキスト		  // surface5 + \b[3]
エ０５．２０２２バ３：テキスト // surface5 + \i[2022] + \b[3]
```

## 接頭辞

ヘッダの前に接頭辞を付けると、改行やウエイトの挙動を変えられます。

| 接頭辞 | 別名 | 効果 | 使いどころ |
|:--|:--|:--|:--|
| `改行無し` | `改行なし`, `改無` | 前の行から改行せず続ける | 同一行内でサーフィスだけ変えたい時 |
| `待ち無し` | `待ちなし`, `待無` | ウエイトを短縮（`\w3`） | テンポよく続ける台詞 |
| `待ちゼロ` | `待０` | ウエイトなし | 瞬時に切り替えたい時 |
| `待ち多め` | `待ち多目`, `待多` | ウエイトを増量（`\w9` x4） | 衝撃の事実の前のタメ |
| `改行多め` | `改行多目`, `改多` | 広めの改行 | 段落の区切り |
| `クリック待ち` | — | `\x`（ユーザーのクリックを待つ） | 長い説明の区切り |
| `改行前` | — | 改行・ウエイトの前にサーフィス変更 | 表情を先に変えてから間を置く |
| `二人` | — | 両キャラ同時発言（`\_s`） | ハモるシーン |
| `同時` | — | 指定キャラ同時発言 | 複数キャラの同時発言 |
| `フォント大` | — | 大きい太字 | 強調 |
| `フォント小` | — | 小さいグレー文字 | 小声・つぶやき |

### 使用例

```yaya
// 感情の切り替えを改行なしで表現
エ０５：やるよ！……
改行無しエ０４：いつか。

// タメの後に衝撃の事実
エ０６：机の上を片付けてたら、3ヶ月前のメモが出てきたの。
テ１０：何て書いてあったの？
エ０４：「重要！あとで絶対やること！」
テ１０：で、何をやるの？
待ち多めエ０６：…
改行無しエ０４．２０３５：何をやるのか書いてなかった。

// クリック待ちで区切る
エ０５：まず前提を説明するね。
クリック待ちエ０５：次に本題です。

// テンポよく畳みかける
エ０５：これがこうなって、
待ち無しエ：こうなって、
待ち無しエ０４：こうなっちゃった。

// 同時発言
二人：バルス！
```

## ユーザー名

トーク内でユーザー名を表示する方法です。

### EX記法内

```yaya
エ０５：（ユーザー）、こんにちは。
```

`（ユーザー）`（全角括弧）がユーザー名に置換されます。`（ユーザ）` `（ゆーざー）` でも可。

`<<' '>>` 内でも動作します。

### SakuraScript/YAYA記法内

```yaya
"\0\s[0]%(username)、こんにちは。\e"
```

## ランダム選択（カッコ記法）

`（A｜B｜C）` と書くと、A, B, C からランダムに1つ選ばれます。

```yaya
エ０５：（おはよう｜こんにちは｜こんばんは）！
```

### 履歴参照

`（H1）` で同じトーク内の最初のランダム選択結果を再利用できます。

```yaya
エ０５：（りんご｜みかん｜ぶどう）がおいしいよね。
テ１０：（H1）いいよね。
```

`（H1）` は1番目の `（）` の結果、`（H2）` は2番目の結果を指します。

## メニュー

### 通常メニュー

```yaya
＠メニュー：表示テキスト｜イベント名
＠メニュー：表示テキスト｜イベント名｜追加引数
```

`｜` で区切り、1つ目が表示テキスト、2つ目以降がイベント名（＝呼び出す関数名）と追加引数。

### 半分メニュー（横2列）

```yaya
＠半分メニュー：選択肢A｜FuncA
＠半分メニュー：選択肢B｜FuncB
```

横2列で並ぶコンパクトなメニューになります。

### 改行多めメニュー

```yaya
＠改行多めメニュー：戻る｜OpenMenu
```

上の項目との間隔が広くなります。

### クリック待ちメニュー

```yaya
＠クリック待ちメニュー：戻る｜OpenMenu
```

この選択肢の直後で `\x`（クリック待ち）が実行されます。

### メニューアイコン

```yaya
＠メニューアイコン：画像ファイル名
＠メニューアイコン：標準		// デフォルトに戻す
```

以降のメニュー項目のアイコンが変わります。

### メニューの実例

```yaya
OpenMenu
{
	<<"
	エ０５：メニューです。
	＠半分メニュー：なにか話して｜Menu_AITALK
	＠半分メニュー：名前を覚えて｜Menu_TELLNAME
	＠半分メニュー：しゃべり頻度｜Menu_TALKINTERVAL
	＠改行多めメニュー：なんでもない｜Menu_CANCEL
	">>
}

Menu_TALKINTERVAL
{
	aitalkinterval = 180
	<<"
	エ０５：3分間隔で喋ってみるわ。
	">>
}
```

## 設定行

`＠設定：` で始まる行は、トークの表示挙動を変更します。

### クイックセクション

```yaya
＠設定：クイックセクション
エ０５：ここはウエイトなしで一気に表示されます。
＠設定：クイックセクション
エ０５：ここからはまた通常速度です。
```

`\_q` のトグル。囲んだ範囲がウエイトなしの高速表示になります。

### パッシブモード

```yaya
＠設定：パッシブモード
```

`\![enter,passivemode]` を挿入。トーク終了後に自動で解除されます。

### タイムクリア

```yaya
＠設定：タイムクリ
```

`\t` を挿入（トーク時間のリセット）。

### タイムアウト

```yaya
＠設定：タイムアウト5000「時間切れ処理」
```

5000ミリ秒（5秒）後に `時間切れ処理` 関数を呼びます。

### チェイン

```yaya
＠設定：チェイン「次のトーク」
```

トーク終了後に `次のトーク` 関数を呼びます（`=====` 区切りの代替手段）。

## 条件分岐

`【条件式】` をヘッダの前に書くと、条件が真の時だけその行が出力されます。

```yaya
エ０５【好感度 >= 50】：仲良くなったね！
エ０５【好感度 < 50】：もっと仲良くなりたいな。
```

### 略記

- `R0`, `R1`... → `reference[0]`, `reference[1]`... の省略形
- `A0`, `A1`... → `_argv[0]`, `_argv[1]`... の省略形

```yaya
エ０５【R0 == 'shell'】：シェルをインストールしたよ。
エ０５【R0 == 'ghost'】：ゴーストをインストールしたよ。
```

### 直前・それ以外

```yaya
エ０５【R0 == 'timeout'】：時間切れだよ。
エ０５【直前】：ここも時間切れの時に出ます。
エ０５【直前否定】：時間切れじゃない時に出ます。
エ０５【それ以外】：上記のどれにも当てはまらない時。
```

- `【直前】` — 直前の条件と同じ結果
- `【直前否定】` — 直前の条件の逆
- `【それ以外】` — それまでの条件が全て偽だった時

### 条件分岐+メニュー

```yaya
＠メニュー【STRLEN(SHIORI3FW.LastAITalk) > 0】：さっきの話は？｜Menu_LASTTALK
＠半分メニュー【'Last' !_in_ R2】：次のページ|
```

メニュー行にも条件を付けられます。条件が偽ならそのメニュー項目は表示されません。

```yaya
エ０５：テキスト
＃ここはコメントです

```

## チェインとランダム選択

トーク内で `=====` や `-----` の区切り線を使えます。

### チェイン（`=====`）

前半のトークが終わった後、自動的に後半のトークが始まります。

```yaya
<<'
エ０５：さて、問題です。
テ１０：なに？
エ０５：日本一高い山は？
=====
エ０６：…
テ１０：富士山でしょ。
エ０５：正解！
'>>
```

### ランダム選択（`-----`）

区切られたブロックからランダムに1つが選ばれます。

```yaya
<<'
エ０５：パターンA。
テ１０：ほう。
-----
エ０５：パターンB。
テ１０：へえ。
-----
エ０５：パターンC。
テ１０：おもしろいね。
'>>
```

## スクリプト直接出力

`＠スクリプト：` でSakuraScriptを直接出力できます。

```yaya
＠スクリプト：\![open,browser,https://example.com]
＠スクリプト：\![change,shell,master]
```

EX記法のトークの途中で、生のSakuraScriptを挿入したい時に使います。

---

## 自動挿入されるもの

EX記法で書くと、以下が自動的に処理されます。

### 句読点ウエイト

| 文字 | 自動挿入 |
|:--|:--|
| `、` `…` | `\w5`（短い間） |
| `。` `！` `？` `」` `』` | `\w9`（長い間） |

手動で `\w` を書く必要はありません。

### スコープ切り替え

キャラが変わると自動的に `\0`, `\1` 等が挿入されます。

### 改行

- 同じキャラの連続行 → `\n`
- キャラ切り替え時 → `\n\n[half]`（やや広い改行）

### ウエイト

- キャラ切り替え時 → `\w9\w9`
- 同じキャラでサーフィス変更あり → `\w9\w9`
- 同じキャラでサーフィス変更なし → `\w9`

### 初期サーフィス

トーク冒頭で、最初に発言するキャラの標準表情が自動設定されます。\0側が先なら `\0\s[0]`、\1側が先なら `\1\s[10]\0` のように。

## SakuraScript記法との混在

あやりりすEXは、返り値がEX記法かSakuraScript記法かを自動判定します。

- ヘッダパターン + `：` or `「` を含む → EX記法として変換
- 複数行ある → EX記法として変換
- それ以外 → SakuraScriptとしてそのまま出力

```yaya
// これはEX記法（自動変換される）
SomeFunc
{
	<<"
	エ０５：テキスト
	">>
}

// これはSakuraScript記法（そのまま出力）
SomeFunc2
{
	"\0\s[0]テキスト\e"
}
```

同じゴースト内でEX記法とSakuraScript記法を混在させることができます。

## 設定ファイル（aya_lilith_ex_config.dic）

### ヘッダ文字の設定

```yaya
// \0側のヘッダパターン（正規表現）
あやりりすEXヘッダ0 {
	'[えエ]'
}

// \1側のヘッダパターン
あやりりすEXヘッダ1 {
	'[てテ]'
}
```

通常使う1文字を `[]` で囲みます。複数の表記（ひらがな・カタカナ等）を許容できます。

サンプルでは `[えエ]`/`[てテ]` を使っていますが、ゴーストのキャラクター名に合わせて自由に変更してください。例えばぽにゃづまでは `[ぽポ]`/`[れレうウばバ]` を使っています。

### 標準表情

```yaya
あやりりすEX標準表情0 {
	0	 // \0側のデフォルトsurface
}

あやりりすEX標準表情1 {
	10	 // \1側のデフォルトsurface
}
```

トーク冒頭でサーフィスを省略した場合にこの値が使われます。

### サーフィス変換

```yaya
あやりりすEXSurfaceTranslate {
	// _argv[0] = 0側/1側
	// _argv[1] = 指定されたサーフィス番号
	// 返り値 = 実際に使うサーフィス番号
	if _argv[0] == 0 {
		if _argv[1] >= 10 && _argv[1] <= 19 {
			_argv[1] - 10
			return
		}
	}
	elseif _argv[0] == 1 {
		if _argv[1] < 10 {
			_argv[1] + 10
			return
		}
	}
	_argv[1]
}
```

スコープとサーフィス番号の不整合を自動修正する関数です。

### SakuraScript文のウエイト支援

```yaya
あやりりすEXSakuraScript文ウエイト支援 {
	0	// 1にするとSakuraScript記法にもウエイト自動挿入
}
```
